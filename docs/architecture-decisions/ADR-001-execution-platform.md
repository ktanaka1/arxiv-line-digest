# ADR-001: 実行基盤に GitHub Actions を採用

## 状況

1日1回・朝に動かすバッチ型 bot の実行基盤が必要。常駐サーバーは不要で、定期実行と
シークレット管理ができればよい。個人運用のため管理コストを最小化したい。

## 選択肢

- **GitHub Actions**: `schedule:` cron で定期実行。シークレットは Repo Secrets。コードと一体。無料枠で十分。
- **Cloud Functions/Run + Cloud Scheduler**: GCP に寄せる（Gemini と同じGCP内）。構成要素が多い。
- **自宅PC/サーバ cron**: 電気代のみだが、マシンが落ちていると動かない。

## 決定

**GitHub Actions**

## 理由

- 1日1回バッチに最適。cron 設定が `schedule:` 一行で済む。
- シークレットもコードも repo 内で完結し、追加のクラウドプロジェクト管理が不要。
- 無料枠で十分賄える規模。
- 実行ログを Actions 画面で確認できる。

## 結果・トレードオフ

- cron 実行に数分の遅延が出ることがあるが、1日1回の用途では許容範囲。
- 通知済み記録を永続化するため、実行後に repo へコミットする必要がある（[ADR-002](ADR-002-datastore.md)）。
  bot コミットには `[skip ci]` を付けて CI ループを防ぐ。
