# ADR-002: 重複排除データを repo 内 JSON のコミットで永続化

## 状況

GitHub Actions の実行環境は毎回まっさらで、ファイルシステムが実行間で永続化されない。
一方で「通知済み arXiv ID」を記録し、再通知を防ぐ必要がある（系統間の再浮上含む）。
記録内容は `arXiv ID / 通知日 / 系統(A/B) / スコア or スター数` 程度で軽量（年間数千行規模）。

## 選択肢

- **repo 内 JSON をコミット**: 実行後に `data/notified.json` を書き戻して `git commit && push`。
- **GitHub Gist**: Gist API で読み書き。repo 本体を汚さない。
- **外部 KV/DB**（Cloudflare KV, Supabase 等）: 拡張性は高いがアカウント・キーが増える。

## 決定

**repo 内 JSON をコミット**

## 理由

- データが軽量で、JSON ファイル1枚で十分。
- 追加サービス・追加アカウントがゼロ。
- 通知履歴が git ログに自然に残り、「いつ何を通知したか」を後から振り返れる。
- データ専用ファイル（`data/notified.json`）に限定すれば、bot コミットによる履歴増加の実害は小さい。

## 結果・トレードオフ

- bot のコミットで main の履歴が伸びる → データ専用ファイルに限定して影響を最小化。
- コミット時は `[skip ci]` を付けて CI の無限ループを防ぐ。
- 将来データが大規模・高頻度化したら外部 KV/DB への移行を再検討する。
